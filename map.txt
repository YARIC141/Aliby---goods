Общее описание
Веб-приложение для отображения загруженной карты (растрового изображения) с геометками, привязанными к заведениям. Карта единственная, загружается в базу данных через API администратором (без интерфейса загрузки в админке). Администратор управляет заведениями и расставляет геометки на карте. Клиент просматривает карту с геометками, видит список заведений под картой и может перейти к детальной информации. Координаты геометок отсчитываются от левого нижнего угла исходного изображения в пикселях.

Роли пользователей

Администратор: загружает карту через API, создаёт/редактирует заведения, управляет геометками на карте (добавляет, удаляет), просматривает карту с геометками.

Клиент: просматривает карту с геометками, нажимает на геометку для просмотра информации, переходит к странице заведения, видит список заведений под картой.

Функциональные требования
3.1. Управление картой (администратор, через API)

Загрузка единственного растрового изображения (PNG/JPEG) через POST-запрос с multipart/form-data. Изображение сохраняется в БД в бинарном поле.

При повторной загрузке старая карта и все связанные геометки удаляются.

В интерфейсе администратора нет формы загрузки карты – только отображение.

3.2. Отображение карты

Карта показывается на общей странице (для клиента и администратора) с возможностью масштабирования (зум колёсиком/кнопками) и панорамирования (перетаскиванием).

Координатная система: (0,0) в левом нижнем углу изображения, ось X вправо, ось Y вверх.

3.3. Управление заведениями (администратор)

Создание, редактирование, удаление записей заведений через интерфейс админки (название, описание, тип кухни, контакты и т.д.).

Список заведений доступен в админке с поиском/фильтрацией.

3.4. Управление геометками (администратор)

Включение режима добавления меток на карте (кнопка «Добавить метку»).

Клик на карте → вычисляются координаты (x, y) → открывается модальное окно: выбор существующего заведения из выпадающего списка ИЛИ создание нового заведения «на лету».

После сохранения геометка привязывается к заведению (place_id) и отображается на карте с названием.

Удаление геометки: при клике на метку в режиме редактирования доступна опция «Удалить метку» (заведение остаётся).

3.5. Просмотр карты и списка заведений (клиент и администратор)

На публичной странице: карта с геометками + под картой список всех заведений.

При клике на геометку – всплывающая панель с краткой информацией (название, тип кухни) и кнопкой «Подробнее».

При клике на элемент списка заведений – переход на страницу детальной информации о заведении.

На странице детальной информации – полные данные заведения.

Варианты использования (Use Cases)

4.1. Загрузка карты (Администратор, через API)

Администратор выполняет POST-запрос к /api/maps с файлом изображения.

Система проверяет тип и размер файла, сохраняет изображение в БД, удаляя предыдущую карту и её геометки.

Успешный ответ содержит id и метаданные карты.

4.2. Создание заведения (Администратор)

Администратор в панели управления заходит в раздел «Заведения», нажимает «Добавить», заполняет форму (название, описание, тип, адрес, телефон и т.д.) и сохраняет.

4.3. Добавление геометки на карту (Администратор)

Предусловие: карта загружена, существует хотя бы одно заведение.

Администратор открывает карту (режим администратора), включает режим «Добавить метку».

Кликает в нужное место карты. Система вычисляет (x,y).

Появляется модальное окно: выбор заведения из списка (поиск) или кнопка «Создать новое заведение».

После выбора/создания нажимает «Сохранить». Система создаёт геометку, карта перерисовывается.

4.4. Просмотр карты и геометок клиентом

Клиент открывает публичную страницу. Система загружает карту и все геометки со связанными названиями заведений.

Геометки отображаются на карте. Клик по метке – всплывающее окно с названием, типом кухни, кнопкой «Подробнее».

Под картой отображается список заведений. Клик по строке – переход на страницу заведения.

4.5. Просмотр деталей заведения

На странице заведения отображаются все сохранённые поля (название, описание, тип, контакты, возможно миниатюра карты с отмеченной геометкой – опционально).

4.6. Удаление геометки (Администратор)

Администратор в режиме просмотра карты (редактирования) кликает по геометке и выбирает «Удалить метку». Запись удаляется, заведение остаётся.

Нефункциональные требования

Веб-приложение с адаптивным интерфейсом для десктопов.

Масштабирование карты: 10%–1000%, панорамирование перетаскиванием.

Все действия с метками – асинхронные (без перезагрузки страницы).

Изображение карты: макс. 20 МБ, поддерживаемые форматы PNG, JPEG.

Максимальное разрешение карты: до 5000×5000 пикселей.

API для загрузки карты и управления метками требуют аутентификации (JWT).

Публичные эндпоинты (получение карты, списка заведений, геометок) доступны без аутентификации.

Модель данных

maps: id, image_data (BLOB), mime_type, width, height, created_at

places: id, name, description, cuisine_type, address, phone, website, created_at, updated_at

markers: id, map_id (FK), place_id (FK), x, y, created_at

users: id, username, password_hash, role (admin/client)

API эндпоинты

Аутентификация:

POST /api/auth/login → JWT

Карта (только для админа):

POST /api/maps (multipart: image)

GET /api/maps/current – метаданные текущей карты

GET /api/maps/current/image – бинарные данные

Заведения:

GET /api/places – список всех заведений (публичный)

GET /api/places/{id} – детали (публичный)

POST /api/places (админ)

PUT /api/places/{id} (админ)

DELETE /api/places/{id} (админ)

Геометки:

GET /api/maps/current/markers – все метки с названиями заведений (публичный)

POST /api/maps/current/markers (админ, тело: {place_id, x, y})

DELETE /api/markers/{id} (админ)

Экраны интерфейса

Публичная страница (клиент и админ в режиме просмотра):

Карта с геометками (зум, панорамирование).

При клике на метку – popup: название, тип кухни, кнопка «Подробнее» (ведёт на страницу заведения).

Под картой список заведений (название, тип). При клике – переход на страницу заведения.

Страница заведения:

Полная информация: название, описание, тип, адрес, телефон, сайт (если есть).

Административная панель:

Раздел «Карта»: та же карта, но с кнопкой «Добавить метку» и возможностью удалять метки по клику.

Раздел «Заведения»: таблица с CRUD-операциями, поиск/фильтр.

Загрузки карты в интерфейсе нет.

Безопасность

Пароли хешированы (bcrypt).

Административные эндпоинты защищены JWT (кроме получения карты/списка/меток).

Валидация загружаемого файла: только изображения, макс. размер.

Экранирование данных при выводе (XSS).
