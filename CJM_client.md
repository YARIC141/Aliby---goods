# CJM — Клиентское приложение Alliby

## Обзор

Клиент — PWA для конечных пользователей: поиск заведений на карте, просмотр меню, покупка абонементов, формирование заказа.

---

## Точки входа

| Вход | Условие | Куда ведёт |
|------|---------|-----------|
| Первое открытие приложения | Нет сессии (`alliby_s` пуст) | → Экран авторизации |
| Повторное открытие | Есть сессия, есть сохранённый экран (`alliby_nav`) | → Последний экран |
| Повторное открытие | Есть сессия, нет сохранённого экрана | → Карта (`screen-map`) |
| Диплинк `#store={id}` | Есть сессия | → Экран заведения |
| Диплинк `#sub={id}` | Есть сессия | → Детальная карточка абонемента |
| Диплинк любой | Нет сессии | → Авторизация → после входа выполнить диплинк |

## Экраны

### 1. Авторизация (`screen-auth`)

**Вкладки:** Войти / Регистрация / Восстановить пароль / Сброс пароля

#### Войти
```
Ввод email + пароль
  → [Войти]
      POST /auth/v1/token?grant_type=password
      ├── Успех → сохранить сессию → navigateDeepLink() → Карта
      └── Ошибка → показать сообщение об ошибке
```

#### Регистрация
```
Ввод email + пароль
  → [Зарегистрироваться]
      POST /auth/v1/signup
      ├── Успех → сохранить сессию → Карта
      └── Ошибка → сообщение об ошибке
```

#### Восстановление пароля
```
Ввод email
  → [Отправить]
      POST /auth/v1/recover
      └── Успех → письмо отправлено (уведомление)
```

#### Сброс пароля
```
Ввод нового пароля (открывается по ссылке из письма)
  → [Сохранить]
      PUT /auth/v1/user
      └── Успех → вкладка "Войти"
```

### 2. Карта (`screen-map`)

**Данные:** GET /rest/v1/stores + GET /rest/v1/rpc/get_stores_with_locations

#### Основные пути
```
Открытие экрана
  → loadMap() → загрузить заведения → отрисовать маркеры на карте
      ├── [Геолокация — кнопка]
      │     navigator.geolocation.getCurrentPosition()
      │     ├── Успех → центрировать карту
      │     └── Отказ → showGeoToast() (тост-уведомление)
      │
      ├── [Поиск по названию/адресу]
      │     filterStores(q) → фильтр списка слева
      │
      ├── [Фильтр по категории заведений]
      │     applyMapFilter() → filterStores() → обновить маркеры/список
      │
      ├── Клик на маркер / элемент списка
      │     showStorePopup(store, listItem)
      │     ├── [Открыть заведение] → goTo('store', store)  ──► Экран заведения
      │     └── [✕ закрыть попап] → closePopup()
      │
      └── [Свернуть/развернуть панель списка]
            toggleMapPanel()
```

### 3. Экран заведения (`screen-store`)

**Данные:** loadStoreSubs() + loadCategories() + loadMenuUserSubs() + loadMenuItems()

#### Основные пути
```
Открытие (goTo('store', storeObj))
  → сохранить S.store → загрузить данные → отрисовать
      │
      ├── [← Назад] → навигация назад (navHistory)  ──► предыдущий экран
      │
      ├── БЛОК АБОНЕМЕНТОВ (store-subs)
      │     Отображает абонементы заведения
      │     ├── Карточка абонемента
      │     │     ├── Если куплен → зелёная цена в карточках товаров
      │     │     └── [Подробнее] → openSubDetailModal(subId)
      │     │           ──► Модал «Детали абонемента»
      │     │
      │     └── [🎫 Подарить] → openGiftConfirm(subId)
      │           ──► Модал «Подтверждение подарка»
      │
      ├── ФИЛЬТРЫ МЕНЮ
      │     ├── Поиск: oninput → debouncedMenuLoad() → loadMenuItems()
      │     ├── Сортировка: onchange → loadMenuItems()
      │     ├── Только в наличии: onchange → loadMenuItems()
      │     └── Категории (кнопки-чипы) → обновить S.menuCatId → loadMenuItems()
      │
      ├── КАРТОЧКА ТОВАРА (item-card)
      │     ├── Если есть купленный абонемент → зелёная цена + название абонемента
      │     ├── Если есть некупленный абонемент → серая подсказка «Абонемент: X ₽»
      │     ├── [В заказ] → addToCartById(id) → updateItemCardInPlace(id)
      │     │     └── Если корзина из другого заведения → confirm() → очистить/отмена
      │     ├── [−][+] (qty-ctrl) → changeQty(id, delta) → updateItemCardInPlace(id)
      │     └── Клик на карточку → openItemDetail(id)
      │           ──► Модал «Детальная карточка товара»
      │
      └── [🛒 Корзина (счётчик)] → goTo('cart')  ──► Экран корзины
```

### 4. Модалы экрана заведения

#### Модал «Детальная карточка товара» (`m-item`)
```
openItemDetail(id)
  → загрузить из itemCache / GET /rest/v1/menu_items?id=eq.{id}
  → показать: фото, название, описание, цена (со скидкой если есть абонемент)
  → Блок абонементов (mi-subs):
      Список абонементов заведения покрывающих этот товар
      ├── Если куплен → badge «Активен», цена со скидкой
      └── Если не куплен → badge «Купить», цена абонемента
  → [В заказ] → addToCartFromDetail() → closeModal + addToCartById()
  └── [✕] → closeModal('m-item')
```

#### Модал «Детали абонемента» (`m-sub-detail`)
```
openSubDetailModal(subId)
  → показать: название, цена, срок, количество использований, описание
  ├── [Купить →] → S.pendingSub = sub → S.pendingGift = false
  │     ──► Модал «Подтверждение покупки» (m-buy)
  ├── [Подарить 🎁] → S.pendingSub = sub → S.pendingGift = true
  │     ──► Модал «Подтверждение покупки» (m-buy)
  └── [✕] → closeModal('m-sub-detail')
```

#### Модал «Подтверждение подарка» (`m-gift-confirm`)
```
openGiftConfirm(subId)
  → предупреждение: срок начнётся с момента активации
  ├── [Подарить] → addGiftSubToCart(subId) → closeModal
  │     └── Абонемент добавлен в giftSubCart
  └── [Отмена] → closeModal('m-gift-confirm')
```

#### Модал «Подтверждение покупки» (`m-buy`)
```
openBuyModal()
  → показать детали абонемента, предупреждение если подарок
  ├── [Купить →] → buySubscription()
  │     POST /rest/v1/user_subscriptions
  │     ├── Успех:
  │     │     → сбросить S.menuUserSubs → loadMenuUserSubs()
  │     │     ├── Если есть товары в корзине → goTo('cart')  ──► Экран корзины
  │     │     └── Иначе → goTo('subscriptions')  ──► Экран абонементов
  │     └── Ошибка → alert
  └── [Отмена] → closeModal('m-buy')
```

### 5. Корзина (`screen-cart`)

**Данные:** renderCart() — при открытии подгружает S.menuStoreSubs + S.menuUserSubs если пусты

#### Основные пути
```
Открытие экрана
  → renderCart()
      │
      ├── БЛОК «КУПИТЬ АБОНЕМЕНТЫ» (subCart)
      │     Абонементы, добавленные к покупке
      │     ├── [✕] → removeFromSubCart(id)
      │     └── [Оформить абонементы] → checkoutSubs()
      │           POST /rest/v1/user_subscriptions (для каждого)
      │           ├── Успех → очистить subCart → уведомление → loadMySubscriptions()
      │           └── Ошибка → alert
      │
      ├── БЛОК «ПОДАРОЧНЫЕ АБОНЕМЕНТЫ» (giftSubCart)
      │     ├── [✕] → removeFromGiftSubCart(id)
      │     └── [Оформить подарки] → checkoutGiftSubs()
      │           POST /rest/v1/user_subscriptions с gift_status='pending'
      │           ├── Успех → очистить giftSubCart → goTo('subscriptions')
      │           └── Ошибка → alert
      │
      ├── БЛОК ТОВАРОВ
      │     Каждый товар:
      │     ├── Если абонемент покрывает → зачёркнутая цена + зелёная цена по абонементу
      │     ├── Если абонемент покрывает частично → «X из Y по абонементу»
      │     ├── Если некупленный абонемент есть → «По абонементу «Название» — X ₽»
      │     └── [−][+] → cartQty(id, delta)
      │
      ├── БАННЕР АБОНЕМЕНТА (если применён)
      │     «✓ Применён абонемент «Название» [к N из M товаров]»
      │
      ├── ИТОГО
      │     ├── Сумма без скидки
      │     ├── Скидка по абонементу (если есть)
      │     └── Итого к оплате
      │
      └── [Оформить заказ] → placeOrder()
            POST /rest/v1/orders
            POST /rest/v1/order_items
            ├── Успех → очистить корзину → уведомление → goTo('home')
            └── Ошибка → alert
```

### 6. Мои абонементы (`screen-subscriptions`)

**Данные:** GET /rest/v1/user_subscriptions (3 запроса: активные, все, подарки)

#### Основные пути
```
Открытие экрана
  → loadMySubscriptions()
      │
      ├── ВКЛАДКИ
      │     ├── «Заведения» (sub-store) — список абонементов от заведений
      │     │     Карточка → openSubDetailModal()  ──► Модал «Детали абонемента»
      │     ├── «Активные» (sub-active) — активные user_subscriptions
      │     │     Карточка → openSubModal(us.id)  ──► Модал «Детальная карточка»
      │     ├── «Все» (sub-all) — все user_subscriptions
      │     └── «Подарки» (sub-gifts) — pending + activated подарки
      │
      ├── [📷 QR] — показать QR-код
      │     ──► Модал «QR-код» (m-qr)
      │
      ├── [Обновить статус абонементов] — перезагрузить данные
      │     → loadMySubscriptions()
      │
      └── АКТИВАЦИЯ ПОДАРКА
            Ввод ID подарочного абонемента
            → [Активировать] → activateGift()
                  POST /rest/v1/rpc/activate_gift_subscription
                  ├── Успех → обновить вкладку «Активные»
                  └── Ошибка → сообщение
```

#### Модал «Детальная карточка абонемента» (`m-usub`)
```
openSubModal(usubId)
  → показать: название, заведение, статус, даты, остаток использований
  → показать условия (time_rules, usage_limits)
  → показать историю списаний (redemption_log)
  └── [✕] → closeModal('m-usub')
```

#### Модал «QR-код» (`m-qr`)
```
showQrModal()
  → сгенерировать QR из S.session.user.id (библиотека qrcodejs)
  → показать: QR-изображение + user ID
  └── [✕] → closeModal('m-qr')
```

**Данные:** GET /rest/v1/rpc/get_user_savings

### 8. Профиль (`screen-profile`)

**Данные:** GET /auth/v1/user + GET /rest/v1/profiles?id=eq.{uid}

```
Открытие экрана
  → loadProfile()
      │
      ├── АККАУНТ
      │     ├── Отображение email, UUID, роли
      │     ├── [Изменить имя] → updateName() → PUT /auth/v1/user
      │     └── [Изменить пароль] → updatePwd() → PUT /auth/v1/user
      │
      ├── НАСТРОЙКИ
      │     ├── [Тёмная/светлая тема] → toggleTheme() → localStorage alliby_theme
      │     ├── [Позиция меню] (мобайл) → toggleNavPos() → localStorage alliby_nav_pos
      │     └── Масштаб шрифта (слайдер) → setFontScale(val) → --fzoom → localStorage
      │
      ├── ЦВЕТА ИНТЕРФЕЙСА
      │     ├── Color-пикеры: акцент, фон, поверхность, паттерн
      │     │     onColorPick(type, hex) → обновить CSS переменные → localStorage
      │     └── [Сбросить цвета] → resetColors() → удалить из localStorage
      │
      └── АККАУНТ (опасные действия)
            ├── [Выйти] → doLogout()
            │     POST /auth/v1/logout
            │     └── Очистить localStorage → goTo('auth')
            └── [Удалить аккаунт] → doDeleteAccount()
                  POST /functions/v1/delete-account
                  └── Очистить localStorage → goTo('auth')
```

## Состояния корзины

```
Корзина пуста
  └── Показать: «Корзина пуста»

Только subCart (абонементы к покупке)
  └── Показать: блок абонементов + «Блюда не добавлены»

Только товары
  └── Показать: товары + итого + [Оформить заказ]

Товары + subCart
  └── Показать: блок абонементов + товары + итого + обе кнопки оформления

Товары + активный абонемент
  └── Показать: товары со скидками + баннер абонемента + скидка в итого
```

## localStorage: сохраняемое состояние

| Ключ | Содержимое | Когда сохраняется |
|------|-----------|------------------|
| `alliby_s` | JWT сессия + refresh token | При входе |
| `alliby_nav` | ID последнего экрана | При каждом переходе |
| `alliby_store` | Объект текущего заведения | При выборе заведения |
| `alliby_cart` | `{storeId, items}` | При изменении корзины |
| `alliby_subcart` | `{subs, gifts}` | При изменении суб-корзины |
| `alliby_theme` | `'dark'` / `'light'` | При смене темы |
| `alliby_nav_pos` | `'right'` / `'bottom'` | При смене позиции меню |
| `alliby_fscale` | Число 1–2 | При изменении масштаба |
| `alliby_colors` | `{accent, bg, surface, pat}` | При выборе цветов |

## Граничные случаи и обработка ошибок

| Ситуация | Обработка |
|---------|----------|
| Корзина из другого заведения | `confirm()` → очистить или отмена |
| Нет сессии при переходе на защищённый экран | `goTo('auth')` |
| Ошибка геолокации | `showGeoToast()` |
| Пустое меню по фильтрам | «Позиций нет» |
| Ошибка загрузки данных | «Ошибка загрузки» |
| Абонемент без скидки (discount_value=0) | Хинт не показывается |
| Абонемент с `remaining_uses=null` | Безлимит (покрывает всё количество) |
| Абонемент покрывает N из M товаров | «(N из M)» в примечании к товару |
| Pull-to-refresh | Перезагружает данные текущего экрана |
