Часть 1: Настройка Базы Данных (Бэкенд)
Первый шаг — подготовить базу данных Supabase для работы с геоданными.

1.1. Включение расширения PostGIS
PostGIS — это расширение для PostgreSQL, добавляющее поддержку геопространственных данных. Включите его в вашем проекте Supabase:

Перейдите в раздел Database → Extensions.

Найдите postgis и включите его. Рекомендуется создать для него отдельную схему, например, gis.

1.2. Создание таблицы для пользовательских меток
Вам понадобится таблица для хранения меток, которые будут создавать пользователи. Выполните этот SQL в SQL Editor вашей Supabase:

sql
-- Включаем расширение, если еще не сделали
CREATE EXTENSION IF NOT EXISTS postgis SCHEMA extensions;

-- Создаем таблицу для меток
CREATE TABLE user_markers (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  geom geometry(POINT, 4326) NOT NULL,  -- Колонка для географической точки
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Создаем пространственный индекс для быстрого поиска по координатам
CREATE INDEX user_markers_geom_idx ON user_markers USING GIST (geom);

-- Включаем Row Level Security (RLS) для контроля доступа
ALTER TABLE user_markers ENABLE ROW LEVEL SECURITY;

-- Политика для вставки (кто угодно может добавлять метки)
CREATE POLICY "Enable insert for all users" ON user_markers
  FOR INSERT WITH CHECK (true);

-- Политика для чтения (все могут видеть все метки)
CREATE POLICY "Enable read access for all users" ON user_markers
  FOR SELECT USING (true);
Важно: Геопространственные колонки в PostGIS имеют особый формат. При вставке данных через API вы будете использовать строковый WKT (Well-Known Text) формат, например 'POINT(-73.946823 40.807416)'. Обратите внимание: сначала указывается долгота (Longitude), затем широта (Latitude), и значения разделяются пробелом, а не запятой.

Часть 2: Подготовка и Загрузка Векторных Карт
Теперь подготовим базовую карту (фоновый слой). Мы используем данные из открытого источника Overture Maps.

2.1. Подготовка данных с помощью Overture Maps
Скачайте данные для нужного региона (например, для Самары) в формате GeoJSON [2†L16-L17] и сгенерируйте из них PMTiles — высокопроизводительный формат для доставки карт по сети [10†L11-L13].

bash
# 1. Установите инструменты
pip install overturemaps
npm install -g osmtogeojson

# 2. Скачайте данные (пример для Самары)
overturemaps download --bbox=50.0,53.1,50.4,53.4 -f geojson --type=building -o samara_buildings.geojson

# 3. Сгенерируйте PMTiles файл с помощью tippecanoe
tippecanoe -o samara_buildings.pmtiles samara_buildings.geojson
Инструмент tippecanoe требует сборки из исходников. Подробные инструкции по установке для вашей ОС можно найти в официальном репозитории проекта.

2.2. Загрузка и публикация в Supabase Storage
В панели Supabase перейдите в раздел Storage и создайте новый бакет (например, maps). Сделайте его публичным, выставив политику доступа на чтение для всех.

Загрузите ваш .pmtiles файл в этот бакет.

Скопируйте публичный URL загруженного файла. Он будет выглядеть примерно так: https://[ваш_проект].supabase.co/storage/v1/object/public/maps/samara_buildings.pmtiles

Часть 3: Разработка Клиентской Части (Фронтенд)
Теперь объединим все на фронтенде.

3.1. Настройка проекта и установка зависимостей
В директории вашего React/Vue/Next.js проекта установите необходимые библиотеки:

bash
npm install @supabase/supabase-js maplibre-gl pmtiles
3.2. Инициализация карты с фоновым слоем (Maplibre + PMTiles)
Создайте компонент карты. Ниже пример инициализации карты с добавлением вашего тайлового слоя из Supabase Storage.

javascript
// mapComponent.jsx
import maplibregl from 'maplibre-gl';
import { Protocol } from 'pmtiles';
import { supabase } from './supabaseClient'; // ваш настроенный клиент Supabase

// Настройка протокола для работы PMTiles
const protocol = new Protocol();
maplibregl.addProtocol('pmtiles', protocol.tile);

const map = new maplibregl.Map({
  container: 'map', // ID HTML-элемента, где будет карта
  style: 'https://demotiles.maplibre.org/style.json', // Базовый минимальный стиль
  center: [50.1242, 53.1959], // [Долгота, Широта] для центра Самары
  zoom: 12
});

map.on('load', () => {
  // Добавляем источник данных из нашего PMTiles файла
  map.addSource('samara-source', {
    type: 'vector',
    url: 'pmtiles://https://[ваш_проект].supabase.co/storage/v1/object/public/maps/samara_buildings.pmtiles'
  });

  // Добавляем слой для отображения зданий
  map.addLayer({
    id: 'samara-buildings',
    type: 'fill',
    source: 'samara-source',
    'source-layer': 'building', // имя слоя внутри PMTiles
    paint: {
      'fill-color': '#FEF08A',
      'fill-opacity': 0.7,
      'fill-outline-color': '#CA8A04'
    }
  });
});
Обратите внимание: для работы PMTiles важен URL с префиксом pmtiles://. Это сигнал для подключенного протокола обрабатывать такой адрес особым образом.

Часть 4: Реализация Пользовательских Функций
4.1. Сохранение геометок с фронтенда
Добавим обработчик клика по карте, который будет сохранять координаты в Supabase.

javascript
// Добавляем обработчик клика на карту
map.on('click', async (e) => {
  // 1. Получаем координаты клика
  const { lng, lat } = e.lngLat;

  // 2. Запрашиваем у пользователя название метки
  const name = prompt('Введите название метки:');
  if (!name) return;

  // 3. Подготавливаем данные для вставки
  // Обратите внимание на формат WKT: 'POINT(longitude latitude)'
  const newMarker = {
    name: name,
    geom: `POINT(${lng} ${lat})`
  };

  // 4. Отправляем данные в Supabase
  const { data, error } = await supabase
    .from('user_markers')
    .insert([newMarker])
    .select();

  if (error) {
    console.error('Ошибка сохранения метки:', error);
  } else {
    console.log('Метка сохранена!', data);
    // Здесь же можно сразу добавить маркер на карту для обратной связи
    new maplibregl.Marker()
      .setLngLat([lng, lat])
      .setPopup(new maplibregl.Popup().setHTML(`<b>${name}</b>`))
      .addTo(map);
  }
});
4.2. Загрузка и отображение всех сохраненных меток
Добавим функцию загрузки всех существующих меток из базы и отображения их на карте при старте.

javascript
// Асинхронная функция загрузки меток
async function loadMarkers() {
  // Выполняем запрос к Supabase. PostGIS-колонку 'geom' нужно преобразовать в понятный формат.
  let { data: markers, error } = await supabase
    .from('user_markers')
    .select('id, name, ST_AsText(geom) as geom_wkt');

  if (error) {
    console.error('Ошибка загрузки меток:', error);
    return;
  }

  markers.forEach(marker => {
    // Парсим строку 'POINT(50.12 53.19)' в массив чисел.
    // Регулярное выражение извлекает числа с плавающей точкой.
    const matches = marker.geom_wkt.match(/[\d.]+/g);
    if (matches && matches.length >= 2) {
      const lng = parseFloat(matches[0]);
      const lat = parseFloat(matches[1]);
      
      new maplibregl.Marker()
        .setLngLat([lng, lat])
        .setPopup(new maplibregl.Popup().setHTML(`<b>${marker.name}</b>`))
        .addTo(map);
    }
  });
}

// Вызываем функцию после загрузки карты
map.on('load', () => {
  // ... (код добавления слоя с зданиями)
  loadMarkers(); // Загружаем пользовательские метки
});
Этот код использует функцию ST_AsText(geom) для преобразования бинарного геометрического формата базы данных в читаемый текст POINT(lng lat).

Часть 5: Поиск по адресу (Геокодирование)
Для поиска мест по названию (например, по адресу) проще всего использовать внешний API, так как это требует поддержки полнотекстового поиска по миллионам названий улиц и городов. Мы добавим поле поиска, которое будет отправлять запросы к бесплатному сервису Nominatim (основан на данных OpenStreetMap).

html
<!-- Добавьте это в HTML, где у вас находится карта -->
<input id="search-input" type="text" placeholder="Поиск адреса в Самаре..." style="position: absolute; top: 10px; left: 50px; z-index: 1000; width: 300px; padding: 8px;"/>
javascript
// JavaScript для обработки поиска
const searchInput = document.getElementById('search-input');

searchInput.addEventListener('keypress', async (e) => {
  if (e.key === 'Enter') {
    const query = searchInput.value;
    if (!query) return;

    // Отправляем запрос к Nominatim API
    const response = await fetch(`https://nominatim.openstreetmap.org/search?q=${encodeURIComponent(query)}&format=json&limit=1`);
    const data = await response.json();

    if (data && data.length > 0) {
      const result = data[0];
      const lng = parseFloat(result.lon);
      const lat = parseFloat(result.lat);
      
      // Перемещаем карту к найденному месту
      map.flyTo({ center: [lng, lat], zoom: 15 });
      
      // Добавляем временный маркер, указывающий на результат поиска
      new maplibregl.Marker({ color: '#3B82F6' })
        .setLngLat([lng, lat])
        .setPopup(new maplibregl.Popup().setHTML(`<b>${result.display_name}</b>`))
        .addTo(map);
    } else {
      alert('Адрес не найден');
    }
  }
});

Заключение
Эта архитектура дает вам несколько ключевых преимуществ:

Эффективность: Базовая карта в формате .pmtiles хранится как единый файл и загружается частями по мере необходимости, экономя трафик. Пользовательские метки хранятся в легковесной базе данных.

Масштабируемость: И Supabase Storage, и Supabase Database автоматически масштабируются под вашу нагрузку.

Гибкость: Вы можете обновлять базовую карту, просто заменяя один .pmtiles файл в Storage, не трогая код приложения.

Прозрачность: Все компоненты (PostGIS, MapLibre, Supabase) являются открытыми и не привязывают вас к проприетарным решениям.

Надеюсь, эта инструкция поможет вам быстро приступить к разработке. Если на каком-то из этапов возникнут вопросы или потребуется уточнить детали, — обращайтесь.
